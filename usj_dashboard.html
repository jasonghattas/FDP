Abstract

This project presents a privacy-preserving machine learning framework that combines Federated Learning (FL) and Differential Privacy (DP). The system trains a predictive model on distributed public health data without centralizing sensitive information.

Federated Learning allows multiple clients to collaboratively train a shared global model while keeping raw data local. Differential Privacy enhances this setup by adding controlled noise during training to guarantee formal privacy protection.

The experimental results demonstrate that the model converges successfully while maintaining strong privacy guarantees. A custom visualization dashboard was also developed to present training progress and results clearly.

1. Introduction

1.1 Background

Machine learning models typically require centralized datasets. However, centralizing sensitive data such as healthcare or financial information introduces major privacy risks.

Federated Learning provides a decentralized approach where training occurs locally on each client device. Differential Privacy strengthens this framework by ensuring that individual data contributions cannot be inferred from the trained model.

1.2 Project Objectives

  • To implement a Federated Learning simulation using distributed clients.
  • To integrate Differential Privacy during local training.
  • To evaluate convergence of the global model.
  • To visualize results using an interactive dashboard.

2. Dataset Description

2.1 Data Source

The dataset used in this project contains smoking-attributable healthcare expenditures across U.S. states.

  • Original dataset: 1560 rows, 19 columns
  • After preprocessing: 1214 rows
  • Selected features: Year, LocationAbbr, Data Value
  • Each state is treated as a separate federated client.

3. System Architecture

3.1 Federated Learning Setup

The system consists of:

  • One central server
  • Five simulated clients (states)
  • Five communication rounds

Training process per round:

  1. The server sends global model parameters.
  2. Clients train locally using their own data.
  3. Differential Privacy is applied during gradient computation.
  4. Updated model parameters are sent back.
  5. The server aggregates updates.

3.2 Differential Privacy Integration

Differential Privacy was implemented using Opacus. Privacy techniques applied:

  • Gradient clipping
  • Gaussian noise addition
  • Privacy budget tracking (ε)

This protects against:

  • Membership inference attacks
  • Model inversion attacks
  • Data leakage risks

6. Visualization Dashboard

A custom HTML dashboard was developed to present:

  • Training rounds
  • Loss curve visualization
  • Privacy metrics
  • Training timestamp

The dashboard dynamically reads data from results.json, allowing live visualization during project demonstration.

7. Discussion

  • No raw data sharing
  • Formal mathematical privacy guarantees
  • Continued model performance and convergence

This approach is applicable to healthcare systems, financial institutions, and distributed IoT networks.

8. Conclusion

This project demonstrates that privacy-preserving distributed machine learning is feasible and effective. By combining Federated Learning with Differential Privacy, it is possible to train models securely without compromising sensitive data.

Future improvements may include:

  • Increasing the number of clients
  • Real multi-device deployment
  • Advanced privacy accounting methods
